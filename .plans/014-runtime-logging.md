# 実行時ログ出力の追加計画

## 目的

実行中の処理状況が分かるように、CLI にログ出力機能を追加する。
ログは標準出力とログファイルの両方に出す。
ログファイルは実行ファイルと同じ場所に `htb2kdl.log` として生成する。

## 修正案

- 実行開始時にログ出力先を初期化する
- 実行開始時に開始日時の区切りバナーを出力する
  - `###############################`
  - `#     2026-06-13 22:48:31     #`
  - `###############################`
- 標準出力と `htb2kdl.log` の両方へ同じログを書き込む
- `htb2kdl.log` は実行ファイルと同じディレクトリに作成する
- `htb2kdl.log` が 5MiB を超える場合はローテーションする
  - `htb2kdl.log.1`
  - `htb2kdl.log.2`
  - `htb2kdl.log.3`
  - 3 世代を超えた古いログは削除する
- `.gitignore` に `*.log` を追加し、生成ログをコミット対象から除外する
- 次の処理の進捗ログを出す
  - はてなブックマーク RSS の取得開始と取得件数
  - キューへの URL 追加と蓄積件数
  - 記事本文の取得開始
  - EPUB 生成開始
  - メール送信開始
- 既存の `generated:`、`queued:`、`sent:`、`removed:` は標準出力のまま維持する
- 既存の警告出力 `warning:` は現状通り標準エラー出力に出す
- README にログ出力とローテーション仕様を追記する
- `internal/cli/cli_test.go` にログ初期化とローテーションのテストを追加する
