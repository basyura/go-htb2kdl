# EPUB 画像ダウンロード対応計画

## 目的

HTML または記事本文に含まれる画像を取得し、生成される EPUB に同梱して、
EPUB リーダー上で画像が表示されるようにする。

## 調査方針

- 既存の変換処理で画像タグをどのように扱っているか確認する。
- EPUB 生成時にリソースを追加する仕組みがあるか確認する。
- テストで現在の画像未表示の原因を再現できる箇所を確認する。

## 修正案

- 本文内の `img` 要素から `src` を抽出する。
- リモート画像をダウンロードし、EPUB 内の画像ファイルとして保存する。
- 本文 HTML の `img src` を EPUB 内の相対パスへ書き換える。
- ダウンロード失敗時は変換全体を壊さない扱いを検討し、既存のエラー方針に合わせる。
- 画像同梱と `src` 書き換えを確認するテストを追加する。

## 確認項目

- `go test ./...` が通ること。
- 生成 EPUB 内に画像リソースが含まれること。
- 本文 HTML から同梱画像への参照になっていること。

## 実施結果

- `book.Write` で章本文の `img src` を解析し、HTTP(S) 画像をダウンロードして EPUB に追加するようにした。
- 相対 URL の画像は章 URL を基準に解決し、本文側の `src` を `images/...` へ書き換えるようにした。
- CLI から EPUB 書き込み処理へ既存の `context` と HTTP クライアントを渡すようにした。
- 画像の同梱、manifest 登録、本文参照の書き換えを確認するテストを追加した。
- `go test ./...` が通ることを確認した。
